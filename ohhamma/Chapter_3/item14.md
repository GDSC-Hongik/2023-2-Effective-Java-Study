# Comparable을 구현할지 고려하라

- Comparable 구현 → 자연적인 순서가 있음
    - 배열 정렬 가능
    - 비교를 활용하는 클래스 : TreeSet, TreeMap, Collections, Arrays
- 순서가 명확한 값 클래스 → 반드시 Comparable 인터페이스 구현하자

## compareTo 메서드 규약

- equals와 유사 : 반사성, 대칭성, 추이성
- compareTo 메서드로 수행한 동치성 테스트 결과 → `equals`와 같아야 함
    - 일관된 순서 결과

## compareTo 메서드 작성 요령

- 입력 인수의 타입을 확인하거나 형변환할 필요x
- 각 필드가 동치인지를 비교하는 것이 아닌, 순서를 비교
- 관계 연산자 `<`와 `>` 사용 → 거추장스럽고 오류 유발, 추천x
- 핵심 필드가 여러 개인 경우, 가장 핵심적인 필드부터 비교하자
    - 비교자 생성 메서드 활용 → 간결한 코드
    - 객체 참조용 비교자 생성 메서드 : 해시코드 값의 차를 기준 → 추이성 위배
        - 대신 정적 compare 메서드 활용하자
